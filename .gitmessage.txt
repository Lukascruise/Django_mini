# 📝 commit template 등록 방법
#############################
1. cd <<.gitmessage.txt이 존재하는 폴더경로로 이동>>
2. git config --local commit.template .gitmessage.txt

#############################
# 🧾 Commit Message Template
#############################

# <타입>: <제목>
#  - 제목은 50자 이내
#  - 무엇을 변경했는지 명확하게 (왜? 보다는 무엇을)
#  - 마침표 금지
#  - 예) ✨feat: 로그인 API 추가

# -------------------------------------
# 본문(선택 사항, 72자 기준 줄바꿈 권장)
#  - 변경 이유, 상세 내용, 추가 정보 등을 작성
#  - 여러 줄일 경우 "-"로 구분
#  - 예)
#    - JWT 기반 로그인 로직 추가
#    - 프로필 이미지 컬럼 추가
# -------------------------------------

# -------------------------------------
# Footer(선택 사항)
#  - 이슈 트래킹 번호 또는 부가 정보
#  - 예) Closes: #101
# -------------------------------------

예시1 : git commit -m "✨feat: Google, Kakao 소셜 로그인 기능 추가" -m "소셜 로그인이 가능하도록 기능을 연결하였습니다. 이로인한 신규파일 및 기존의 로그인 파일의 수정이 존재합니다."
예시2 : git commit
✨feat: 사용자 로그인 기능 추가
(한 줄 생략합니다.)
이제 사용자는 이메일과 비밀번호를 사용하여 계정에 로그인할 수 있습니다.
여기에는 성공의 경우와 실패의 동작 모두 포함됩니다.
로그인에 실패 할 경우 사용자에게 오류 메시지를 표시해줍니다.

- 백엔드에 인증 로직을 구현했습니다.
- 프론트엔드에 로그인 양식 구성 요소를 만들었습니다.
- 잘못된 자격 증명 및 네트워크 오류와 같은 엣지 케이스를 처리했습니다.

Issues: #1234

#############################
# Commit Type Guide
#############################

- ✨feat:새로운기능추가
- 🐛fix:기능 수정, 버그수정
- 💡chore:오타 수정 및 새로운 기능이 추가되지 않고, 코드가 변경 된 경우 (주석 추가 및 수정 포함)
- 📝docs:문서수정 (readme 수정 시)
- 🚚build : 빌드 관련 파일 수정 및 삭제한 경우(파일 및 폴더 수정 또는 삭제)
- ✅test:테스트코드,리팩터링테스트코드추가(프로덕션코드변경X)
- ♻️refactor:코드리팩터링
- 🚑hotfix : 긴급 수정
